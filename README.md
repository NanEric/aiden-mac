# Aiden macOS Local Integration

This repository contains a runnable baseline for:
- `AidenTrayMac` (menu bar UI process)
- `AidenRuntimeAgent` (background supervisor/API process)
- runtime dependency tooling that downloads and verifies `otelcol` and `victoria-metrics` for local runs.

## Runtime layout
- Runtime dependencies:
  - `~/Library/Application Support/Aiden/runtime/bin/otelcol`
  - `~/Library/Application Support/Aiden/runtime/bin/victoria-metrics-prod`
- Runtime config/artifacts (generated by Tray bootstrap):
  - `~/Library/Application Support/Aiden/runtime/collector/config/collector.yaml`
  - `~/Library/Application Support/Aiden/runtime/deps.lock.json`
- Runtime LaunchAgent (generated at runtime):
  - `~/Library/LaunchAgents/com.aiden.runtimeagent.plist`

The tray process bootstraps runtime config and LaunchAgent files at startup.

## Build
```bash
swift build
```

## Test
```bash
swift test
```

## Runtime dependency installation
`install-runtime-deps.sh` (bundled into the release installer) installs `otelcol` and `victoria-metrics` binaries under `~/Library/Application Support/Aiden/runtime/bin`.
Collector config and runtime dependency lock are generated by Tray bootstrap at startup.

Note: if upstream artifacts are unsigned on macOS, configure `download-collector.sh`/`download-vm.sh` to skip team checks and rely on SHA256 pinning instead.
